Java注释和运行原理

详述Java注释

为了方便注明程序的功能与开发人员调试，现代计算机语言都有着注释功能。

单行注释与块注释

  • 行注释

    对一行文字进行注释，只需在注释文本前加上两个斜杠//

  • 块注释（多行注释）

    对一段需要换行的长文本进行注释需要在注释以/*开头，以*/结尾。

Java运行原理

一个java文件的大致运行流程是这样的：

graph LR; Java源文件--编译-->Java字节码文件; Java字节码文件--运行-->结果;

字节码文件不同于二进制文件，二进制文件是可以被计算机直接执行的。当对字节码执行编译的java命令时，字节码文件就会被编译成二进制文件。

具体流程：

graph LR; 将字节码加载到内存-->校验字节码 ;

Java标识符

基本要求：

  1. Java标识符必须由字母(大小写均可)，数字，下划线，$符号组成。
  2. 不能以数字开头。
  3. 没有长度限制。
  4. 不能是关键字。

规范要求：

  1. 见名知义，命名规范
  2. 大小写规范

### Java基本数据类型

整数类-byte,short,int和long

这四种都可表示有符号的整数，区别在于二进制位数不同，也就是范围不一样。

  • byte 数据类型是8位、有符号的，以二进制补码表示的整数；

    最小值是 -128（-2^7）；

    最大值是 127（2^7-1）；

  • short 数据类型是 16 位、有符号的以二进制补码表示的整数

    最小值是 -32768（-2^15）；

    最大值是 32767（2^15 - 1）；

  • int 数据类型是32位、有符号的以二进制补码表示的整数；

    最小值是 -2,147,483,648（-2^31）；

    最大值是 2,147,483,647（2^31 - 1）；

  • long 数据类型是 64 位、有符号的以二进制补码表示的整数；

    最小值是 -9,223,372,036,854,775,808（-2^63）；

    最大值是 9,223,372,036,854,775,807（2^63 -1）；

小数类-double,float

  • double 数据类型是双精度、64 位、符合IEEE 754标准的浮点数；

    浮点数的默认类型为double类型；

  • float 数据类型是单精度、32位、符合IEEE 754标准的浮点数；

    直接使用float浮点数时，要在数字后加上f(不区分大小写)

    float 在储存大型浮点数组的时候可节省内存空间；

字符型-char

  • char类型是一个单一的 16 位 Unicode 字符；

    最小值是 \u0000（即为 0）；

    最大值是 \uffff（即为 65535）；

可以直接对char赋值一个整数，该整数代表字符对应的ASCII码。

对于特殊字符，要用转义字符代替：

布尔型-boolean

  • boolean数据类型表示一位的信息；

    只有两个取值：true(真)和false(假)；

    默认值是 false
